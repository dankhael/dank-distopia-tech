# Guia para Iniciantes em Programação

## 1. Aprender Inglês
Antes de entrar na área é super importante aprender inglês. Todo o material e partilha de informação é realizado em inglês (documentação, aulas, termos, etc.). Pode existir muito material em português, mas quanto mais fundo forem, mas difícil será de encontrar.

**Dicas:**
- Se não souber nada, deve tentar pegar um curso na net (vídeos, duolingo, na escola, etc.)
- Para quem percebe o mínimo tentar ver vídeos/filmes com legendas em inglês
- O mais importante é sempre tentarem consumir em inglês (livros, vídeos, música, etc.)

## 2. Ter noção das diferentes áreas
É normal estar perdido no começo e não saber o que especializar. Mas tenta estudar aquilo que gostas. Vai sempre haver conceitos chatos ou momentos que vais sentir perdido, desta forma tenta estudar aquilo que estás a gostar para não desanimar. Existe milhares de áreas de atuação/especialização (no server temos algumas, mas não engloba um décimo do que existe) tenta explorar um pouco de tudo e caso estejas a gostar dos conceitos, explora. A chave para o sucesso é seres bom em alguma coisa e não seres o que sabe um pouco de tudo e não conseguires fazer nada.

## 3. Nem tudo é Programar

Mergulhar na tecnologia não significa apenas escrever código para o Back-end ou Front-end. O sucesso de um projeto depende de um ecossistema de áreas que trabalham em conjunto, e ter uma noção básica sobre elas vai tornar-te um profissional muito mais completo.

### Bancos de Dados e a Estrutura da Informação
Antes de qualquer linha de código ser funcional, precisamos de saber onde e como guardar a informação. É aqui que entra a gestão de bases de dados, utilizando tecnologias SQL ou NoSQL para garantir que os dados do projeto estão organizados, seguros e acessíveis.

### Design e Experiência: UI e UX
Um software pode ser tecnicamente perfeito, mas se for difícil de usar, ninguém o usará. O UI (User Interface) foca-se naquilo que vemos: a disposição dos elementos e a estética da interface. Já o UX (User Experience) foca-se no que sentimos: a fluidez da navegação, o número de cliques para chegar a um objetivo e como o utilizador interage com o produto de forma comercial e natural.

### Pesquisa de Campo e Feedback
Não desenvolvemos para nós próprios, mas sim para pessoas. A pesquisa de campo é o que nos permite entender as necessidades reais do público. Isto envolve desde questionários direcionados até ao lançamento de versões Alpha para recolha de feedback. Conversar com testadores e analisar dados de uso é o que permite ajustar o rumo do projeto antes do lançamento final.

### Gerenciamento de Projeto e Engenharia de Software
Um projeto sem organização raramente chega ao fim. Esta área trata da maquinaria por trás do desenvolvimento: organização de tarefas, definição de prazos e documentação. O foco aqui é aplicar técnicas de produção para garantir que as funcionalidades são entregues a tempo e que a equipa comunica de forma eficiente.

### Ciência de Dados e Inteligência Artificial
Ao contrário da programação tradicional, estas áreas vivem muito da matemática: estatística, álgebra linear, cálculo, probabilidade e lógica. Um cientista de dados utiliza estas ferramentas para interpretar grandes volumes de informação e criar modelos preditivos. Aqui, o código funciona como um meio para extrair insights e resolver problemas complexos em contextos como saúde, finanças ou marketing.

### Nota final
Saber lógica e saber programar é a base que evita que portas se fechem por falta de conhecimento técnico. No entanto, não precisas de ser um especialista em tudo. O mais importante é ter a sensibilidade de entender como cada uma destas peças se encaixa para construir algo maior. A programação abre caminhos, mas é o conhecimento transversal que te permite percorrê-los com sucesso.


## 4. Lógica de programação
Ao iniciar o aprendizado em programação, antes mesmo de escolher uma linguagem, é necessário ter em mente que a coisa mais importante é a lógica para solucionar o problema fornecido.

Comece realizando exercícios simples (https://exercicios.dunossauro.com/01_estrutura_sequencial/) até se sentir mais confortável com desafios mais complexos.

Una bem o estudo teórico com a prática, mantenha uma rotina resolvendo exercicíos de Leet Code (https://leetcode.com/)

- BroCode: https://www.youtube.com/@BroCodez/videos

## Dicas de Software

### Visual Studio Code
Editor de texto integrado com um ambiente de desenvolvimento de software, existem outros bons IDEs, mas normalmente é para tasks/áreas específicas, mas o vscode é o mais abrangente.
https://code.visualstudio.com

### GitHub
Plataforma de gerenciamento de repositorios git, é importante terem uma conta e guardarem os vossos projetos. Para os informáticos o GitHub é obrigatório para realizar essa tarefa. Aprender git também é algo super importante.

### Tentar ter e usar Linux
Muito software só funciona em Linux e maior parte das vezes, as ferramentas e as bibliotecas/frameworks funcionam melhor, além disso, usuários de Linux são incentivados a interagir com o computador através do terminal. Existem diferentes métodos para terem Linux do vosso computador sem ter que mandar fora o Windows:

**Métodos de instalação:**
- **WSL**: Basicamente um emulador de linux, não é o mais fácil de usar, mas é o mais fácil de instalar e super prático pra quem não quer sair do Windows (https://www.youtube.com/watch?v=Wdy-Q_1PGgY)
- **Dual boot**: É muito prático e conseguem manter a performance. Conceito é terem na inicialização do seu computador a escolha que sistema operacional querem começar. Não é o mais profissional, mas é muito útil, principalmente para quem começa. Convém ter muito espaço no computador (https://www.youtube.com/watch?v=KWVte9WGxGE)
- **Virtual Machines**: É dos mais profissionais e é super importante para tarefas que podem estragar o sistema operativo. Convém ter bastante RAM e Memória (https://www.youtube.com/watch?v=nvdnQX9UkMY)
- **Docker**: Software pra empacotar aplicações em ambientes controlados, super leve e profissional. Não é simples de usar, mas é essencial para tarefas mais avançadas. Saber Docker é extremamente relevante para teres no currículo (https://www.youtube.com/watch?v=eGz9DS-aIeY)

**Distribuições recomendadas:**
- **Ubuntu**: Super user friendly para quem está habituado com o Windows e gosta de usar o ambiente gráfico com janelas flutuantes. Para começar, talvez seja a melhor opção (https://ubuntu.com)
- **Debian**: Mais leve que o Ubuntu e super interessante para quem só quer programar sem ocupar muitos recursos do próprio computador (https://www.debian.org)
- **Arch Linux**: Muito usado pelos programadores devido à ao sistema de janelas. Ele é bastante útil para caso não queiram usar o mouse e mais o teclado. Não recomendo usar para quem é iniciante (https://archlinux.org)

Existem outras distribuições muito boas, mas são mais específicas para certas tarefas/funções, de qualquer forma, sinta se livre pra escolher uma distribuição Linux do seu interesse.

### Ter um software/caderno para documentar
Muitas vezes vai ficar horas a aprender e a pesquisar sobre algo e eventualmente vai se esquecer. É super importante documentares o teu raciocínio/pesquisas. Existem milhares de softwares para diferentes gostos. Eu gosto de apontar algoritmos e raciocínios em caderno, mas o digital têm sempre a vantagem de ferramenta de pesquisa e de outras features que são limitadas pelo papel.

**Softwares bons para documentar:**
- **Notion**: Aplicação de notas, organização e gerenciamento de projetos, super intuitivo e profissional. Muito útil para trabalho em equipe
- **Obsidian**: Aplicação de notas muito completa e com vários plugins que você pode adicionar dependendo da sua necessidade (https://obsidian.md/)
- **Excalidraw**: Muito bom pra documentar/explicar ideias abstratas e desenhar esquemas, pode ser utilizado pelo navegador através do site (https://excalidraw.com/) ou até como plugin no obsidian
- **Miro/Figma/Trello**: Sistema de boards interativas, os vossos esquemas ficam parecidos ao esquema mental do Cellbit para explicar Evangelion

## Dicas Gerais

### 1. Tutorial Hell
Não tentes ficar sempre a ouvir tutoriais e mais tutoriais e achares que estás a aprender. E mesmo que faças os exercícios propostos nos tutorias, isso NÃO CHEGA. Só vais aprender com muitas horas na prática e a bater com a cabeça na parede. Tenta sempre fazer projetos de pequena/média escala e para consolidar os conteúdos. O importante é sempre compreender a lógica das ferramentas e dos conceitos para eventualmente aplicares no futuro.

### 2. Usar o server para tirar dúvidas
Usem o canal de dúvidas para qualquer coisa. É normal ficarem perdidos sobre algum conceito ou ficarem sem rumo. Usem e abusem do fórum de dúvidas do server, muitas dúvidas podem parecer bobas mas é melhor perguntar do que ficar sem saber, é claro que coisas muito simples é melhor pesquisar no google. Na eventualidade de existir algum conceito com muitas dúvidas, podemos marcar sessões por vídeo chamada para ajudar todos ao mesmo tempo. O importante é interagir com a comunidade para apoiar uns ao outros.

### 3. Participar nas atividades do server
Como referido anteriormente, é importante engajar com a comunidade e consolidar os conteúdos aprendidos. Temos a ambição de realizar diferentes atividades com a comunidade para engajar, às quais é necessária a vossa participação. Desta forma, é essencial a vosso empenho e dedicação para que o projeto cresça quer em conteúdo quer em desenvolvimento interpessoal. O vossos feedback e participação são de extrema importância.

### 4. Ter sempre a mente aberta para novas tecnologias
Às vezes, um projeto dá mais certo que outro, não porque tem o melhor código ou organização, mas por ter simplesmente o melhor conjunto de tecnologias que acabam por levar a melhor performance e usabilidade. Com isto, fiquem atentos a tecnologias emergentes e líderes de mercado. Alguns devs acabam por adotar certas tecnologias como time de futebol e acabam utilizando-a pra tudo e como diz o ditado, quando você está com um martelo na mão, tudo parece um prego.

### 5. Evitar LLMs/Chat Bots para resolver exercícios/fazer código
LLMs e Chat Bots são muito bons para esta área, mas no começo evitem usar os chat bots para resolver exercícios. Pode ser interessante usarem as LLMs para resolver erros de compilação ou até mesmo para ajudar na pesquisa. Mas usem com responsabilidade.
