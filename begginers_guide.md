2. Ter noção das diferentes áreas
   É normal estar perdido no começo e não saber o que especializar. Mas tenta estudar aquilo que gostas. Vai sempre haver conceitos chatos ou momentos que vais sentir perdido, desta forma tenta estudar aquilo que estás a gostar para não desanimar.
Existe milhentas áreas de atuação/especialização (não server temos algumas, mas não engloba um décimo do que existe) tenta explorar um pouco de tudo e caso estejas a gostar dos conceitos, explora. A chave para o sucesso é seres bom em alguma coisa e não seres o que sabe um pouco de tudo e não conseguires fazer nada.

4. Lógica de programação
Ao iniciar o aprendizado em programação, antes mesmo de escolher uma linguagem, é necessário ter em mente que a coisa mais importante é a lógica para solucionar o problema fornecido.
Comece realizando exercícios simples ( https://exercicios.dunossauro.com/01_estrutura_sequencial/ ) até se sentir mais confortável com desafios mais complexos.
Una bem o estudo teórico com a prática, mantenha uma rotina resolvendo exercicíos de Leet Code. ( https://leetcode.com/ )
Alguns canais no Youtube para iniciantes
BroCode - https://www.youtube.com/@BroCodez/videos
FreeCodeCamp - https://www.youtube.com/@freecodecamp/videos

Durante as aulas você pode escrever e executar códigos Online neste site ( https://onecompiler.com/python )


5. Cada área tem o seu roadmap 
É difícil de dizer quais são as melhores tecnologias para começar para uma pessoa que não sabe que especializar. O importante é que cada especialização tem um conjunto de tecnologias e matérias que é essencial conhecer que eventualmente cruzam com outras áreas. Não hesitem em questionar sobre o que devem estudar e explorem os conteúdos e atividades disponíveis no servidor.

Dicas de Software:
	- Visual Studio Code (https://code.visualstudio.com)
	Existem outros bons IDEs, mas normalmente é para tasks/áreas específicas, mas o 	vscode é o mais abrangente. 

	- GitHub 
	É importante terem uma conta e guardarem os vossos projetos. Para os informáticos o 	GitHub é obrigatório para realizar essa tarefa. Aprender git também é algo super 	importante.

	- Tentar ter e usar Linux:
	Muito software só funciona em Linux e maior parte das vezes, as ferramentas e as bibliotecas/frameworks funcionam melhor. Existem diferentes métodos para terem Linux do vosso computador sem ter que mandar fora o Windows:
		- WSL -> Não é o mais fácil de usar, mas é o mais fácil de instalar e super prático (https://www.youtube.com/watch?v=Wdy-Q_1PGgY)
		- Dual boot -> É muito prático e conseguem manter a performance. Conceito é terem na inicialização do vosso computador a escolha que sistema operativo querem começar.
Não é o mais profissional, mas é muito útil, principalmente para quem começa. Convém ter muito espaço no computador (https://www.youtube.com/watch?v=KWVte9WGxGE)
		- Virtual Machines -> É dos mais profissionais e é super importante para tarefas que podem estragar o sistema operativo. Convém ter bastante RAM e Memória. (https://www.youtube.com/watch?v=nvdnQX9UkMY)
		- Docker -> Super leve e profissional. Não é simples de usar, mas é essencial para tarefas mais avançadas. Saber Docker é extremamente relevante para teres no currículo. (https://www.youtube.com/watch?v=eGz9DS-aIeY)
		
	Quanto a que distribuição que devem usar, deixo aqui algumas recomendações:
		-> Ubuntu: Super user friendly para quem está habituado com o Windows e gosta de usar o ambiente gráfico com janelas flutuantes. Para começar, talvez seja a melhor opção (https://ubuntu.com);
		-> Debian: Mais leve que o Ubuntu e super interessante para quem só quer programar sem ocupar muitos recursos do próprio computador (https://www.debian.org)
		-> Arch Linux: Muito usado pelos programadores devido à ao sistema de janelas. Ele é bastante útil para caso não queiram usar o rato e mais o teclado. Não recomendo usar para quem é iniciante (https://archlinux.org)

	Existem outras distribuições muito boas, mas são mais específicas para certas tarefas/funções.

	- Ter um software/caderno para documentar:
		Muitas vezes vais ficar horas a aprender e a pesquisar sobre algo e eventualmente vais te esquecer. É super importante documentares o teu raciocínio/pesquisas. Existem milhentos softwares para diferentes gostos. Eu gosto de apontar algoritmos e raciocínios em caderno, mas o digital têm sempre a vantagem de ferramenta de pesquisa e de outras features que são limitadas pelo papel. Eis alguns softwares bons para documentar:

Dicas Gerais:
	1. Tutorial Hell
	Não tentes ficar sempre a ouvir tutorias e mais tutorias e achares que estás a aprender. E mesmo que faças os exercícios propostos nos tutorias, isso NÂO CHEGA. Só vais aprender com muitas horas na prática e a bater com a cabeça na parede. Tenta sempre fazer projetos de pequena/média escala e para consolidar os conteúdos. O importante é sempre compreender a lógica das ferramentas e dos conceitos para eventualmente aplicares no futuro.

	2. Usar o server para tirar dúvidas
	Usem o canal de dúvidas para qualquer coisa (sem javardice). É normal ficarem perdidos sobre algum conceito ou ficarem sem rumo. Usem e abusem do fórum de dúvidas do server. Na eventualidade de existir algum conceito com muitas dúvidas, podemos marcar sessões por vídeo chamada para ajudar todos ao mesmo tempo. O importante é interagir com a comunidade para apoiar uns ao outros.

 	3. Participar nas atividades do server
	Como referido anteriormente, é importante engajar com a comunidade e consolidar os conteúdos aprendidos. Temos a ambição de realizar diferentes atividades com a comunidade para engajar, às quais é necessária a vossa participação. Desta forma, é essencial a vosso empenho e dedicação para que o projeto cresça quer em conteúdo quer em desenvolvimento interpessoal. O vossos feedback e participação são de extrema importância.
	

 	4. Ter sempre a mente aberta para novas tecnologias:
	Às vezes, um projeto dá mais certo que outro, não porque tem o melhor código ou organização, mas por ter simplesmente o melhor conjunto de tecnologias que acabam por levar a melhor performance e usabilidade. Com isto, fiquem atentos a tecnologias emergentes e líderes de mercado.

	5. Evitar LLMs/Chat Bots para resolver exercícios/fazer código
	LLMs e Chat Bots são muito bons para esta área, mas no começo evitem usar os chat bots para resolver exercícios. Pode ser interessante usarem as LLMs para resolver erros de compilação ou até mesmo para ajudar na pesquisa. Mas usem com responsabilidade.
