# GUIA DE APRESENTAÇÃO CYBERSECURITY

## EXPLICAÇÃO DA ÁREA
**Definição:** A cibersegurança (Cybersecurity) é a prática de proteger sistemas, redes, programas e dados contra ataques digitais, acessos não autorizados, danos ou roubo. É uma área que combina conhecimentos de informática, matemática (especialmente teoria dos números e álgebra para criptografia), engenharia e análise de ameaças. A cibersegurança é essencial em todas as indústrias, desde saúde e finanças até governos e infraestruturas críticas, protegendo informação sensível e garantindo a continuidade de operações. Abrange múltiplas camadas: segurança de aplicações (web, mobile, desktop), sistemas operativos, redes, criptografia e certificados digitais (SSL/TLS, PKI), gestão de identidades e acessos, segurança cloud, e práticas de DevSecOps. Uma pessoa que atua em cibersegurança precisa de ter uma grande compreensão de como os sistemas funcionam, como as vulnerabilidades são exploradas, e como implementar defesas eficazes para proteger contra ameaças em constante evolução.


**Termos Chave:** Criptografia, SSL/TLS, PKI, Penetration Testing, Firewalls, Zero Day, Malware, Ethical Hacking, CTF (Capture The Flag), CVE (Common Vulnerabilities and Exposures)

## PROFISSÕES E ATUAÇÃO

### Responsabilidades
A atuação na cibersegurança está fortemente ligada ao desenvolvimento de software seguro e sistemas de proteção. Os profissionais trabalham para proteger aplicações desde a sua conceção, identificar vulnerabilidades através de testes técnicos, e desenvolver ferramentas que defendem infraestruturas digitais.


As responsabilidades incluem:
- Desenvolver código seguro e fazer code reviews para identificar vulnerabilidades;
- Implementar sistemas de segurança e proteção (WAF, rate limiters, API gateways, DDoS mitigation, criptografia, SSL/TLS, OAuth, JWT);
- Realizar pentesting, bug bounty hunting e desenvolver ferramentas de segurança;
- Integrar SAST/DAST e dependency scanning em pipelines CI/CD.

### Impacto no Projeto
A cibersegurança é essencial para a qualidade e viabilidade de qualquer produto de software. Proteger aplicações contra vulnerabilidades evita data breaches, mantém a confiança dos utilizadores e garante compliance regulatório. Integrar segurança desde o início do desenvolvimento previne custos elevados de correção tardia. Implementar sistemas de proteção robustos (WAF, rate limiting, encryption, authentication) diferencia produtos profissionais de soluções amadoras e protege tanto a empresa como os utilizadores finais.

- **Python:** Scripting de segurança, automação de testes e desenvolvimento de exploits
- **JavaScript:** Compreender e testar vulnerabilidades web (XSS, CSRF)

### Ferramentas de Pentesting
- **Burp Suite:** Web application security testing - https://portswigger.net/burp
- **Nmap:** Network discovery e port scanning - https://nmap.org/
- **Metasploit:** Framework de penetration testing - https://www.metasploit.com/
